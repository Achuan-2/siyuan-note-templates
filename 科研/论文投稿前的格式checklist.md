## Manuscript

### 整体结构与一致性

- [ ] **标题大写格式是否统一**：各章节标题要么都是首字母大写，要么是全大写

  - 首字母大写：Data availability
  - 全大写：Data Availability
- [ ] **缩写词首次定义问题**：

  - [ ] **首次定义**: 所有缩写词在首次出现时都已给出全称并加上括号，例如 `Signal-to-noise ratio (SNR)`。
  - [ ] **全文统一使用**: 定义后，全文统一使用缩写形式（如 `SNR`），避免全称和缩写混用。
  - [ ] **独立定义**: **摘要 (Abstract)**  和 **正文 (Main Text)**  被视为独立部分。因此，一个缩写词如果在摘要中定义过，在正文中首次出现时**仍需重新定义**。
- [ ] **统一描述**：检查对同一概念、方法、设备或物质的描述是否在全文中保持一致。不要前文用一个术语，后面又改成另一个描述

  - 错误示例：前面用"patient"，后面用"subject"
  - 错误示例：前面用"real-time PCR"，后面用"RT-PCR"
- [ ] **时态一致性 (Consistent Tense)** :

  - [ ] **引言 (Introduction)** : 通常使用现在时描述背景知识和研究目的。
  - [ ] **方法 (Methods)**  和 **结果 (Results)** : 描述已完成的实验和观察到的结果，通常使用**过去时**。
  - [ ] **讨论 (Discussion)**  和 **结论 (Conclusion)** : 解释结果的意义和展望未来，通常使用现在时和过去时结合。
- [ ] **Figure legends**

  - [ ] figure legends格式统一，风格统一
  - [ ] 清晰说明误差棒 (error bars) 代表的是标准差 (SD) 还是标准误 (SEM)。
- [ ] **图表/公式引用风格统一**：Fig. 1、Extended Data Fig. 1
- [ ] **统计报告完整一致：** 检验名、n、均值±误差、p 值、显著性标注

  - [ ] 报告中包含：**检验名称**（如 `unpaired two-tailed t-test`）、**统计量值**（如 `t(18) = 2.34`）、**自由度/样本量**（如 `n = 10`）、**精确p值**（如 `p = 0.026`，而非 `p < 0.05`，除非 `p < 0.001`）。
  - [ ] 均值±误差的表示一致（如 `mean ± SEM`）。
  - [ ] 显著性标注符号统一（如 `*` for `p < 0.05`, `**` for `p < 0.01`, `***` for `p < 0.001`）。

### 文本与排版细节

- [ ] **英式或美式拼写统一，例如**colour vs color，programme vs program
- [ ] **字体统一：** 使用Time News Roman，注意插入某些字符，可能会变为中文字体，需要改为英文字体
- [ ] **空格**

  - [ ] **数值与单位**: 数值和单位之间必须有空格。

    - **错误**: `30s`, `100μm`, `15mL`，`90 °`
    - **正确**: `30 s`, `100 μm`, `15 mL`, `90°`

    **例外**：

    - 角度（`°`）、角分（`'`）、角秒（`"`）前不加空格。
    - **百分号**「%」**千分号**「‰」，根据 SI **标准**应该**添加空格** ，但在大多数论文中也常不加空格 ，可自行取舍。
  - [ ] **数学运算符**: 加减乘除、约等号、正负号（`+`, `-`, `×`, `÷`, `≈`, `±`）前后应有空格。

    - **示例**: `6.4% ± 1.7%`, `p < 0.01`, `a + b = c`
  - [ ] **括号**: 括号与括号外的文字之间应有空格，但与括号内的文字之间不应有空格。

    - **示例**: `The result (Fig. 1a) clearly shows a significant increase.`
  - [ ] **冗余空格**: 使用“查找-替换”功能，将所有“连续两个空格”替换为“一个空格”，清理全文。
- [ ] **连字符 (Hyphen) 与连接号 (En Dash)** :

  - [ ] **连字符 (-)** : 用于复合词，如 `state-of-the-art`, `high-performance`。
  - [ ] **En Dash (–)** : 用于表示数值范围，比连字符稍长。en-dash 在Word中可通过 `Ctrl + Num-`（数字键盘）或插入符号获得

    - **错误**: `1-4 mW`, `pages 10-15`
    - **正确**: `1–4 mW`, `pages 10–15`
- [ ] **单位与符号** 

  - [ ] **大小写**: 检查单位大小写是否正确。

    - **错误**: `mw` , `kwh`，<sup>`Mhz`</sup>
    - **正确**: `mW`, `kWh`，`MHz`
  - [ ] **特殊字符**: 确保使用正确的特殊字符，而不是用普通字母替代。

    - **错误**: `um` (微米), `uL` (微升)
    - **正确**: `μm`, `μL`
  - [ ] **标准格式**: 确保单位格式符合国际标准，如 `m/s` 或 `m·s⁻¹`。
- [ ] **数字**

  - [ ] **千位分隔符**: 对于大于999的数字，根据期刊习惯使用千位分隔符（逗号）。

    - **示例**: `1,300`
  - [ ] **句首数字**: 避免用阿拉伯数字作为句子的开头，应将其拼写出来或改写句子。

    - **错误**: `10 samples were analyzed.`
    - **正确**: `Ten samples were analyzed.` 或 `We analyzed 10 samples.`
  - [ ] 范围数字保留相同小数位：1.20-1.45（非1.2-1.45）
- [ ] **斜体**

  - [ ] **拉丁语**: *in vivo*, *in vitro*, *in situ* 等应为斜体。
  - [ ] **统计学符号**: *p* (p-value), *n* (样本量), *t* (t-test), *F* (F-statistic), *r* (相关系数) 等应为斜体。

    - **示例**: *p* < 0.05, *n* = 10
  - [ ] **生物学命名**: 物种的属名和种名应为斜体，如 *Escherichia coli*。基因用斜体，蛋白质用正体。
  - [ ] **变量符号**: 文中使用的数学或物理变量通常使用斜体。

### 参考文献 (References)

- [ ] **核查文献信息是否正确**：即使使用文献管理软件（如 EndNote, Zotero）插入文献后，仍需**手动核对**文献信息是否准确（作者、年份、标题、期刊名、卷、期、页码），因为可能爬取错误。
- [ ] **检查文献标题的大小写：** 是否符合期刊要求（`Sentence case` vs. `Title Case`）。有些软件抓取时会变成全大写，需要手动修改。
- [ ] **遵循期刊格式**: 对照目标期刊的 "Guide for Authors"，检查引文格式的每一个细节，如期刊名是全称还是缩写、是否有 `et al.`、作者名后的逗号、年份的括号等。
- [ ] **页码与年份区间使用 en dash**：132–145，2018–2020
- [ ] **文中引用与列表对应**: 确保正文中引用的每一篇文献都出现在参考文献列表中，确保文末列表**没有**未被引用的文献。
